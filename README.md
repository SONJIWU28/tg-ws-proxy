# TG WS Proxy 

**Локальный MTProto WebSocket proxy для Telegram Android**, переработанный под более стабильную работу на смартфонах: с корректным MTProto handshake, re-encryption потока, prewarmed WebSocket pool, TCP fallback и более живучей фоновой работой.

Это не просто мобильная обёртка над старой SOCKS-логикой. Проект был **архитектурно переделан** так, чтобы Android-версия работала ближе к сильным сторонам desktop-референса и лучше справлялась с переподключениями, сменой сети и длительными передачами данных.

> [!IMPORTANT]
> Эта версия ориентирована именно на **MTProto proxy-схему**, а не на классический SOCKS5-режим.
> Подключение к Telegram выполняется через **MTProto secret** и локальный адрес приложения.

> [!NOTE]
> Проект вдохновлён оригинальной идеей `tg-ws-proxy` и развитием WebSocket/MTProto.  
> Отдельное упоминание и уважение автору оригинального проекта — **[Flowseal](https://github.com/Flowseal)**.

## 🌟 Что реализовано в этой версии

Функции управления вынесены в удобный Android-интерфейс, а транспортный слой переработан под более правильную MTProto-модель.

- **MTProto WebSocket bridge:** приложение работает как локальный MTProto proxy, а не как простая SOCKS5-обёртка.
- **Корректный secret-based handshake:** используется `MTProto secret`, совместимый со схемой подключения Telegram.
- **Re-encryption client ↔ relay:** трафик пере-шифровывается на нужном уровне транспорта, как и ожидается для MTProto bridge.
- **Prewarmed WebSocket pool:** заранее прогретые WebSocket-соединения уменьшают задержки и улучшают устойчивость.
- **TCP fallback:** если WebSocket-ветка деградирует, приложение может использовать резервную TCP-схему.
- **DC:IP configuration:** можно явно задавать список датацентров и IP-адресов.
- **Локальный host и port:** настраиваемый локальный адрес прокси, например `127.0.0.1:1443` или `127.0.0.1:1080`.
- **Генерация MTProto secret:** secret можно быстро сгенерировать прямо из интерфейса.
- **Интеграция с Telegram:** кнопка **«Добавить в Telegram»** формирует подключение в формате MTProto proxy.
- **Foreground Service:** сервис держится в фоне заметно стабильнее, чем у обычных фоновых задач.
- **Улучшенная устойчивость к смене сети:** при переключении Wi-Fi / mobile network транспортное состояние сбрасывается корректнее.
- **Лучшая работа с большими файлами:** архитектура переделана так, чтобы приложение не так легко “сыпалось” на длинных и тяжёлых передачах.
- **Снижение проблем с reconnect:** переподключения больше не должны выглядеть как бесконечная паническая атака сокетов.
- **Более правильная мобильная архитектура:** проект приближен по логике к сильной desktop-версии, а не просто “похож внешне”.

## ⚙️ Как это работает

```text
Telegram Android
    ↓
MTProto proxy link
    ↓
Локальный proxy на устройстве (host:port + secret)
    ↓
MTProto bridge
    ↓
WebSocket pool / TCP fallback
    ↓
Telegram DC
```
